# FanhB Mobile — Claude Code Context

## What this project is
iOS couples app (drawing + nudges + memory vault). SwiftUI + UIKit Metal canvas, realtime via WebSocket, WidgetKit + NotificationServiceExtension sharing state over an App Group.

## Xcode project management
The project uses **XcodeGen** — never edit `FanhB.xcodeproj` manually.

```bash
# Regenerate after editing project.yml or adding new files/targets
xcodegen generate

# Install xcodegen if missing
brew install xcodegen
```

Entitlements and Info.plist files are **generated by xcodegen** from `project.yml`. Edit the `entitlements.properties` and `info.properties` sections in `project.yml`, then regenerate.

## Adding a new Swift Package
1. Create `Packages/<Layer>/<Name>/Package.swift` and `Sources/<ModuleName>/` with at least one `.swift` file.
2. Add to the `packages:` section in `project.yml`.
3. Add as a dependency under the relevant target(s) in `project.yml`.
4. Run `xcodegen generate`.

## Architecture rules (enforced by package boundaries)
- Domain packages must not import SwiftUI, UIKit, or URLSession.
- Feature packages (Phase 1+) depend only on their corresponding Domain + Core packages.
- Cross-feature navigation lives in `App/` (Coordinators).
- View models are `@MainActor`; domain use cases are `nonisolated`.
- All DTOs and domain models must be `Sendable`.
- Tokens live only in Keychain (`kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly`).
